# Contributing to R2PQ

Thank you for considering contributing!

## How to Contribute
1. Fork the repository  
2. Create a feature branch  
3. Write clear code with comments  
4. Add tests for cryptography-critical changes  
5. Submit a pull request with a clear description  

## Code Style
- Follow existing directory structure  
- Use descriptive commit messages  
- Keep PRs focused on one change at a time  

## Tests
All cryptographic or consensus-critical code must include tests.
This ensures safety, reproducibility, and long-term reliability.

## Documentation
If you add or modify features, update the docs in `r2pq-docs`.

## Communication
Questions? Open a Discussion or Issue.
